공지사항

  • DevOn Rich UI를 프로젝트에 적용하려면 적용 절차를 준수해주십시요. (진행 절차 가이드 참조)
  • 개발자는 개발환경을 준수하여 개발하여야 합니다. 자신의 스타일로 개발할 경우 에러에 대한 추적이 어려워질 수 있습니다.
  • Q&A는 아키텍트가 취합하여 요청하시기 바랍니다.
  • 오류가 발생할 경우에는 웹 표준 오류인지? 서버 오류인지? RUI 오류인지를 파악한 후 RUI 오류에 대해서만 요청 바랍니다. 그외에는 다른팀에 요청하세요.
  • RUI는 프로젝트의 일반적인 요구사항이 있는 기능과 유틸리티(prototype 기능 포함) 등이 갖춰져 있습니다. 따른 프레임웤과 혼용하여 사용하지 마십시요.
  • 프로젝트 진행시에는 웹표준에 맞춰 개발하셔야 하며, 가장 이슈 사항으로는 html dom에 id를 중복해서 사용할 경우 문제를 추적하기가 어렵습니다.
  • firefox나 다른 브라우져에서는 정상적으로 처리되는데 ie에서만 문제가 발생할 경우 javascript에서 ','가 추가로 들어가 있는지 확인후 그래도 문제가 발생하면 요청해주십시요.
  • 개발자는 웹표준 및 ajax를 최소한 이해를 해야 합니다.
  • RUI의 버그로 인한 에러일 경우 방어 코딩을 하지 마시고 문제점에 대해서 해결 요청을 하고 정상적인 처리 방법으로 코딩하시기 바랍니다. (진행중 RUI의 이슈사항이 발생 경우에도 동일)
  • 신규 개발자가 투입 될 경우 반드시 api 보는 방법과 샘플을 찾는 방법에 대해서는 교육을 받아야 합니다. 기존 아키텍트분이 최소한의 교육을 해주시기 바랍니다.

개발 필수 문서

  • 개발 가이드
  • 최신 소스
  • RUI 사용 진행 절차 가이드
  • 디자인 가이드
  • 서버 샘플 소스(DevOn 기준)

개발자 기술 필수 이해

  • 웹표준
  • html/css/javascript
  • ajax
