Connection Manager singleton 패턴은 비동기화 transaction들을 만들고 관리하기 위한 method를 제공한다. /rui/sample/general/base/connectSample.html
지정한 URL로 비동기형 요청을 실행하며, 그 성공 또는 실패 결과를 지정한 callback function으로 받는다.
sample: URL과 parameters를 지정하며 비동기 요청한 후 그 성공 또는 실패의 결과를 callback function으로 받는다.
var url = '../sample/data/data.json'; var params = 'filter=A1'; var request = Rui.LConnect.asyncRequest('GET', url, { success: function(e){ //success console.log(e.responseText); }, failure: function(e){ //failure } }, params);
지정한 URL로 동기형 요청을 실행하며, 그 성공 또는 실패 결과를 지정한 callback function으로 받는다. 동기형 요청시 일부 브라우저는 요청의 결과가 callback 되기 까지 브라우저의 상태를 잠금으로 인하여 그동안 수행되어야할 다른 프로세스들이 중지되므로 성능 저하가 발생될 수 있으므로 주의하여 사용해야함.
sample: URL과 parameters를 지정하며 동기 요청한 후 그 성공 또는 실패의 결과를 callback function으로 받는다.
var url = '../sample/data/data.json'; var params = 'filter=A1'; var request = Rui.LConnect.syncRequest('GET', url, { success: function(e){ //success console.log(e.responseText); }, failure: function(e){ //failure } }, params);
비동기 방식을 이용하여 서버로 파일등이 포함된 FORM전송을 할 때 사용할 수 있다.
sample: 서버로 지정된 FORM 전송을 위해 비동기 요청 전 FORM을 지정하기 위해 setForm()을 사용한다.
var form = document.getElementById('form1'), url = '../sample/data/targetServlet'; Rui.LConnect.setForm(form); Rui.LConnect.asyncRequest('POST', url, { success: function(e){ //success console.log(e.responseText); }, failure: function(e){ //failure } });
