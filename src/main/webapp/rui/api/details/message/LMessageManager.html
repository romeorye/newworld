다국어 메시지를 관리하는 기능 /rui/resources/locale/lang-XXXX.js 경로에 정의된 메시지값을 국가에 따라서 다국어 자동으로 로딩해서 메시지를 출력한다. 단, 프로젝트의 모든 메시지를 lang-XXX.js에 넣어서 처리할 수 도 있지만 javascript의 특성상 페이지 자체가 무거워지므로 일반적인 프로젝트 메시지 서버의 메시지 처리 방식을 사용하길 권장한다. DevOn의 경우 커스텀 태그 사용
sample: 메시지를 읽어 오는 기본 샘플
alert(Rui.getMessageManager().get('$.base.msg100'));
현재 셋팅되어 있는 Locale 정보를 변경한다. 해당 로케일을 설정할 경우에는 lang-로케일.js 파일이 locale 디렉토리에 존재해야 한다.
sample: 메시지의 국가를 'en_US'로 설정한다.
var mm = new Rui.message.LMessageManager(); mm.setLocale('en_US');
메시지 데이터를 직접 추가한다.
sample: test 메시지를 추가한다.
var localeData = { locale : 'ko_KR', message:{test:'테스트'} } var mm = new Rui.message.LMessageManager(); mm.addLocaleData(localeData);
메시지 데이터를 읽어 온다.
sample: 국가별 메시지를 추가한후 메시지를 출력한다.
var mm_Ko = new Rui.message.LMessageManager(); mm_Ko.load({locale:'ko_KR'}); alert(mm_Ko.get('$.core.test')); var mm_En = new Rui.message.LMessageManager({locale:'en_US'}); var newData_ko_KR = { locale:'ko_KR', message : { test2:'한글' } } var newData_en_US = { locale:'en_US', message : { test2:'english' } } mm_Ko.addLocaleData(newData_ko_KR); mm_En.addLocaleData(newData_en_US); var data = mm_Ko.get('$.message.test2'); alert(data); var data = mm_En.get('$.message.test2'); alert(data);
